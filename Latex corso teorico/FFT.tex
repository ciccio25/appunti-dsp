\section{Costo computazionale della DFT}

\footnote{

    Latex | 2025-11-10 Mattino | pag 201

Per una sequenza $x \left[ n\right]$ di lunghezza N, come scritto in \ref{sec: DFT - Discrete Fourier Transform} \nameref{sec: DFT - Discrete Fourier Transform} a pag \pageref{sec: DFT - Discrete Fourier Transform}, 
possiamo scrivere la sua DFT come: 

{
    \Large 
    \begin{equation}
        X \left[ k\right]
        = 
        \sum_{n = 0}^{N - 1}
        x \left[ n\right]
        \cdot 
        W_N^{kn}
        \quad 
        \text{ per }
        k = 0, 1, \dots, N - 1
    \end{equation}
}

dove $W_N$ prende il nome di Twiddle ed è uguale a: 

{
    \Large 
    \begin{equation}
        W_N 
        = 
        e^{\frac{- \jmath 2 \pi}{N}}
    \end{equation}
}

I problemi del calcolo della DFT sono i seguenti, per ogni valore di k: 

\begin{itemize}
    \item è una sommatoria di prodotti 
    \item ci sono N moltiplicazioni complesse per ogni valore di k 
    \item ci sono $N - 1$ somme complesse 
\end{itemize}

ovvero, calcolando la complessità del calcolo, avremo che: 

\begin{itemize}
    \item le moltiplicazioni complesse sono dell'ordine di $N^{2}$ ($4 N^{2}$ reali)
    \item le somme complesse sono dell'ordine di $N \left(N - 1\right)$ ($N \left(4N - 1\right)$ reali)
\end{itemize}

\begin{tcolorbox}
Le moltiplicazioni complesse sono dell'ordine $4 N^{2}$ perchè, se ipotizziamo $x\left[ n\right]$, allora avremo che, per generici valori a, b, c, d: 

{
    \Large 
    \begin{equation}
        \begin{split}
            \left(a + \jmath b\right)
            \cdot 
            \left(c + \jmath d\right)
            &= 
            a c + \jmath ad + \jmath bc + \jmath^{2}bd
            \\
            &=
            a c + \jmath ad + \jmath bc -bd
        \end{split}
    \end{equation}
}

che, come si può vedere, sono 4 moltiplicazioni per ogni termine, quindi complessità $4 N^{2}$. 
\end{tcolorbox}

Considerando N molto elevato, la complessità del calcolo di una DFT è di O($N^{2}$), cioè una complessità molto elevata. \newline 

\begin{tcolorbox}
    Il concetto di complessità di un algoritmo in realtà dovrebbe essere stato affrontato nei corsi precedenti di informatica, 
    come ad esempio "Elementi di Informatica" e/o, nel mio caso, per l'esame di "Programmazione ad oggetti". \newline 

    Per un breve ripasso su cosa si intende per l'ordine di grandezza di complessità di un algoritmo, vi lascio i seguenti video: 

    \begin{itemize}
        \item  Big-O Notation in 100 Seconds by Fireship \\\url{https://www.youtube.com/watch?v=g2o22C3CRfU}
        \item Big-O Notation - For Coding Interviews by NeetCode \\\url{https://www.youtube.com/watch?v=BgLTDT03QtU}
    \end{itemize}
\end{tcolorbox}

Il Twiddle (cioè l'esponenziale) gode di due proprietà fondamentali: quella di simmetria e quella di periodicità. \newline 

Ad esempio possiamo scrivere che: 

{
    \Large 
    \begin{equation}
        W_{N}^{kN}
        = 
        W_{N}^{0}
        = 
        1
    \end{equation}
}


perchè: 


{
    \Large 
    \begin{equation} \label{eq: simmetria e periodicità del Twiddle}
        W_{N}^{kN}
        = 
        W_{N}^{0}
        = 
        1
        \leftrightarrow
        \begin{cases}
            W_{N}^{k \left(N - n\right)} 
            = 
            W_{N}^{- k  n} 
            = 
            \left( W_{N}^{k  n}  \right)^{*}
            \\
            W_{N}^{k \left(N + n\right)} 
            = 
            W_{N}^{\left(k + N\right)  n} 
            = 
            \left( W_{N}^{k  n}  \right)
        \end{cases}
    \end{equation}
}

$W_{N}^{k \left(N - n\right)} = W_{N}^{- k  n} = \left( W_{N}^{k  n}  \right)^{*}$ spiega la proprietà di simmetria del Twiddle. \newline

$W_{N}^{k \left(N + n\right)} = W_{N}^{\left(k + N\right)  n} = \left( W_{N}^{k  n}  \right)$ spiega la proprietà di periodicità del Twiddle. \newline 

Grazie a queste due proprietà del Twiddle, esistono diversi tipi di algoritmi per calcolare la DFT con un costo computazionale inferiore ad O($N^{2}$) che prendono il nome di FFT (acronimo di Fast Fourier Transform). \newline

Ipotizzando una sequenza $x \left[ n\right]$ di lungheezza N pari, allora possiamo scomporre la sequenza $x \left[ n \right]$ in due sottosequenze di lunghezza $\dfrac{N}{2}$, 
contenenti rispettivamente i campioni con indici pari e dispari, come nel seguente caso: 

{
    \Large 
    \begin{equation}
        \begin{split}
        X \left[ k\right]
        &= 
        \sum_{n = 0}^{N - 1}
        x \left[ n\right]
        \cdot 
        W_{N}^{kn}
        \\
        &= 
        \sum_{n \text{ pari}}
        x \left[ n\right]
        \cdot 
        W_{N}^{kn} 
        + 
        \sum_{n \text{ dispari}}
        x \left[ n\right]
        \cdot 
        W_{N}^{kn}       
        \end{split}
    \end{equation}
}

allora, possiamo calcolare la DFT di ogni sottosequenza come: 

\newpage 

{
    \Large 
    \begin{equation}
        \begin{split}
        X \left[ k\right]
        &= 
        \sum_{n \text{ pari}}
        x \left[ n\right]
        \cdot 
        W_{N}^{kn} 
        + 
        \sum_{n \text{ dispari}}
        x \left[ n\right]
        \cdot 
        W_{N}^{kn}       
        \\
        &= 
        \sum_{m = 0}^{\frac{N}{2} - 1}
        x \left[ 2m\right]
        \cdot 
        W_{N}^{k2m} 
        + 
        \sum_{m = 0}^{\frac{N}{2} - 1}
        x \left[ 2m + 1\right]
        \cdot 
        W_{N}^{k \left( 2m + 1\right)} 
        \end{split}
    \end{equation}
}

ma, grazie alle proprietà degli esponenziali, possiamo scrivere che: 

{
    \Large 
    \begin{equation}
        W_{N}^{k \left(2 m + 1\right)}
        = 
        W_{N}^{k} \cdot W_{N}^{k 2m}
    \end{equation}
}

in cui a sua volta possiamo scrivere: 

{
    \Large 
    \begin{equation}
        W_{N}^{k 2 m}
        = 
        \left( W_{N}^{2} \right)^{km}
        = 
        W_{\frac{N}{2}}^{km} 
    \end{equation}
} 

\begin{tcolorbox}
Ricordando che il twiddle $W_N$ vale: 

{
    \Large 
    \begin{equation}
        W_N = e^{\frac{-\jmath 2 \pi}{N}}
    \end{equation}
}

allora possiamo scrivere che: 

{
    \Large 
    \begin{equation}
        \begin{split}
            W_N^{k2m} 
            &= 
            e^{\frac{-\jmath 2 \pi}{N} k 2 m}
            = 
            e^{\frac{-\jmath 4 \pi}{N} k m}
            \\
            &\updownarrow
            \\
            \left( W_{N}^{2} \right)^{km}
            &= 
            e^{\left(2 \cdot \frac{-\jmath 2 \pi}{N}\right) k  m} 
            = 
            e^{\left(\frac{-\jmath 4 \pi}{N}\right) k  m} 
            \\
            &\updownarrow
            \\
            W_{\frac{N}{2}}^{km}
            &=
            e^{\left(- \jmath \frac{2 \pi}{\frac{N}{2}} \right) km}
            = 
            e^{\left(- \jmath 2 \pi \cdot \frac{2}{N} \right) km}
            = 
            e^{\left( \frac{- \jmath 4 \pi}{N} \right) km}
        \end{split}
    \end{equation}
}

cioè, i tre twiddle $W_N^{k2m}$, $ \left( W_{N}^{2} \right)^{km}$, $W_{\frac{N}{2}}^{km}$ sono tutti equivalenti 
\end{tcolorbox}

Grazie a questa proprietà del Twiddle, possiamo ulteriormente semplificare $X \left[ k\right]$ come: 

{
    \Large 
    \begin{equation}
        \begin{split}
        X \left[ k\right]       
        &= 
        \sum_{m = 0}^{\frac{N}{2} - 1}
        x \left[ 2m\right]
        \cdot 
        W_{N}^{k2m} 
        + 
        \sum_{m = 0}^{\frac{N}{2} - 1}
        x \left[ 2m + 1\right]
        \cdot 
        W_{N}^{k \left( 2m + 1\right)} 
        \\
        &\downarrow
        \\
        X \left[ k\right]       
        &= 
        \sum_{m = 0}^{\frac{N}{2} - 1}
        x \left[ 2m\right]
        \cdot 
        W_{\frac{N}{2}}^{km} 
        + 
        W_{N}^{k}
        \cdot
        \sum_{m = 0}^{\frac{N}{2} - 1}
        x \left[ 2m + 1\right]
        \cdot 
        W_{\frac{N}{2}}^{k m} 
        \end{split}
    \end{equation}
}

            \\

ma, dalle proprietà del Twiddle, sappiamo che:

Se la lunghezza di $x \left[ n\right]$ originale è una potenza di 2, cioè: 

{
    \Large 
    \begin{equation}
        N = 2^{M}
    \end{equation}
}

allora, si può dividere la DFT fino a $M - 1$ volte, utilizzando la stessa struttura di calcolo. \newline 

\begin{tcolorbox}
    Sotto a questo albero di 2 punti, non si può ulteriormente dividere. 
\end{tcolorbox}

Ad esempio, se: 

{
    \Large 
    \begin{equation}
        N = 8 = 2^{3}
        \to 
        M = 3
    \end{equation}
}

Quindi possiamo dividiere in 3 stadi: 

{
    \Large 
    \begin{equation}
        \text{DFT 8 punti}
        \to
        \text{DFT 4 punti}
        \to
        \text{DFT 2 punti} 
    \end{equation}
}

{
    \Large 
    \begin{equation}
        \begin{cases}
            r = 2^{M - l}
            \\
            p^{''} = p^{'} + 2^{l - 1}
        \end{cases}
    \end{equation}
}

con i valori di $q^{'}$, $q^{''}$, $p^{'}$, $p^{''}$ opportuni in base allo stadio in cui si svolge la DFT. \newline

\subsection{Radix 2 DIT FFT in place}

\footnote{ 
    Slide del prof | 8 - FFT 1 | pag 13 - 14 \\
    Appunti| 2025-11-04 | pag  4 \\ 
    Latex | 2025-11-10 Mattino | pag 209

In base alle formule ottenute, in ogni stadio occorre valutare $\dfrac{N}{2}$ butterflies. \newline 

Poichè ogni butterfly è indipendente dalle altre (cioè per indipendente si intende che si richiede 2 dati e se ne produce altri 2 senza interazioni), 
allora l'ordine di valutazione delle $\dfrac{N}{2}$ butterflies di ogni stadio è arbitrario, 
purchè le espressioni analitiche siano rispettate. \newline 

Grazie a questa proprietà di indipendenza, ci sono perciò numerose forme realizzative diverse, a seconda di come si ordinano i dati in ogni stadio. \newline 

Se si indica con $X^{l}$ per $k = 0, 1, \dots, N - 1$  il vettore dei dati allo stadio l per $k = 0, 1, \dots, M$, allora risulta che: 

{
    \Large 
    \begin{equation}
        X^{0} \left[ k\right] 
        = 
        x \left[ n\right]
    \end{equation}
}

dove $x \left[ n\right]$ è la sequenza di ingresso. \newline 

\begin{tcolorbox}
Questa relazione è valida grazie al bit reversing che si fa tra bot di ingresso e di uscita nella Radix 2 DIT FFT in place, 
quindi è possibile fare questa associazione.     
\end{tcolorbox}

\begin{tcolorbox}[colback=green!5!white,colframe=green!75!black]
TO-DO: Spiegare perchè si usa il bit-reversing in questo tipo di FFT
\end{tcolorbox} 

Invece: 

{
    \Large 
    \begin{equation}
        X^{M} \left[ k\right] 
        = 
        X \left[ n\right]
    \end{equation}
}

dove $X \left[ n\right]$ è la sequenza di uscita. \newline 

Allora, il modo più naturale di disporre i dati è quello di far corrispondere alla posizione $X^{l - 1}$ dello stadio $l - 1$, 
la posizione dello $X^{l} \left[ k\right]$ allo stadio l come si vede nella seguente figura: 

\begin{figure}[h]
    \centering
    \includegraphics[scale = 0.35]{Radix 2 DIT FFT stadi in place.png}
\end{figure} 

dove: 

\begin{itemize}
    \item $q = p + 2^{l - 1}$
    \item $r^{'} = 2^{M-1} \cdot \left(p\right)_{2^{l-1}}$
    \item $r^{''} = r^{'} + \dfrac{N}{2}$
\end{itemize}

quindi: 

{
    \Large 
    \begin{equation}
        W_N^{r^{''}} = - W_N^{r^{'}}
    \end{equation}
}

\section{Costo computazionale Radix-2 FFT}

\footnote{ 
    Slide del prof | 8 - FFT 2 | pag 13 - 14\\
    Appunti| 2025-11-04 | pag  5  \\
    Latex | 2025-11-10 Mattino | pag 214 
} 

Considerando il grafo di una FFT DIT Radix-2: 

\begin{figure}[h]
    \centering
    \includegraphics[scale = 0.35]{Radix 2 DIT FFT grafo.png}
\end{figure} 

in cui: 

{
    \Large 
    \begin{equation}
        \begin{cases}
            X^{l} \left[ p^{'}\right] = X^{l - 1} \left[ q^{'}\right] + X^{l - 1} \left[ q^{''}\right] \cdot W_{N}^{r} 
            \\
            X^{l} \left[ p^{''}\right] = X^{l - 1} \left[ q^{'}\right] - X^{l - 1} \left[ q^{''}\right] \cdot W_{N}^{r}  
        \end{cases}
    \end{equation}
}

Invece considerando il grafo di una FFT DIF Radix-2: 

\begin{figure}[h]
    \centering
    \includegraphics[scale = 0.35]{Radix 2 DIF FFT grafo.png}
\end{figure} 

in cui: 

{
    \Large 
    \begin{equation}
        \begin{cases}
            X^{l} \left[ p^{'}\right] = X^{l - 1} \left[ q^{'}\right] + X^{l - 1} \left[ q^{''}\right] 
            \\
            X^{l} \left[ p^{''}\right] = \left( X^{l - 1} \left[ q^{'}\right] - X^{l - 1} \left[ q^{''}\right] \right)\cdot W_{N}^{r}  
        \end{cases}
    \end{equation}
}

Come si può notare dalle figure, i grafi della DIT e della DIF sono diversi, ma svolgono le stesse operazioni perchè, 
per ogni butterfly, c'è: 

\begin{itemize}
    \item 1 prodotto complesso 
    \item 2 somme complesse 
\end{itemize}

\begin{tcolorbox}
Generalamente avremo a che fare con $x \left[ n\right]$ reali, quindi non complesse. \newline 

In questo caso consideriamo somme complesse perchè consideriamo $x \left[ n\right]$ complessa come caso peggiore.     
\end{tcolorbox}

Generalizzando il caso a una FFT a N punti, 
avremo che ogni butterfly calcola 2 punti dove: 

\begin{itemize}
    \item per ogni stadio di N punti è necessario $\dfrac{N}{2}$ 
    \item se si svolge una FFT a N punti dove $N = 2^{M}$, allora ci saranno M stadi 
\end{itemize} 

\newpage 

Quindi, il costo computazionale di una FFT Radix-2 è dovuto a: 

\begin{itemize}
    \item $\dfrac{N}{2} \cdot M$ prodotti complessi 
    \item $N \cdot M$ somme complesse 
\end{itemize}

quindi O($N \cdot \log_{2} \left(N\right)$). \newline 


\begin{tcolorbox}
    Questo perchè l'algoritmo FFT Radix-2 può essere anche visto come un algoritmo ad albero. \newline 

    Per visualizzarlo meglio puoi vedere questi video in ordine:

\section{Calcolo della convoluzione e della correlazione}

\footnote{ 
    Slide del prof | 8 - FFT 2 | pag 20 - 21\\
    Appunti| 2025-11-05 | pag 1 - 2  \\ 
    Latex | 2025-11-10 Mattino | pag 216 
} 

Date due sequenze reali $x_1 \left[ n\right]$ e $x_2 \left[ n\right]$ possiamo scrivere che la convoluzione si può calcolare come: 

{
    \Large 
    \begin{equation}
        x_1 \left[ n\right] * x_2 \left[ n\right] 
        = 
        \sum_{m = -\infty}^{+ \infty}
        x_1 \left[ m\right] 
        \cdot 
        x_2 \left[ n - m\right]  
    \end{equation}
}

dove, nel dominio DTFT avremo che è uguale a: 

{
    \Large 
    \begin{equation} 
        \sum_{m = -\infty}^{+ \infty}
        x_1 \left[ m\right] 
        \cdot 
        x_2 \left[ n - m\right]
        \leftrightarrow 
        X_1 \left(e^{\jmath \omega}\right)
        \cdot  
        X_2 \left(e^{\jmath \omega}\right)
    \end{equation}
}

Invece la correlazione tra due sequenze reali $x_1 \left[ n\right]$ e $x_2 \left[ n\right]$ si calcola come: 

{
    \Large 
    \begin{equation}
        \begin{split}
        x_1 \left[ n\right] * x_2 \left[ -n\right] 
        &= 
        \sum_{m = -\infty}^{+ \infty}
        x_1 \left[ m\right] 
        \cdot 
        x_2 \left[ n - \left(-m\right)\right]
        \\
        &=
        \sum_{m = -\infty}^{+ \infty}
        x_1 \left[ m\right] 
        \cdot 
        x_2 \left[ n + m\right]       
        \end{split}
        \end{equation}
}

dove, nel dominio DTFT avremo che è uguale a: 

{
    \Large 
    \begin{equation} 
        \sum_{m = -\infty}^{+ \infty}
        x_1 \left[ m\right] 
        \cdot 
        x_2 \left[ n + m\right]
        \leftrightarrow 
        X_1 \left(e^{\jmath \omega}\right)
        \cdot  
        X_2^{*} \left(e^{\jmath \omega}\right)
    \end{equation}
}

La differenza tra la formula della convoluzione e correlazione è che:

\begin{itemize}
    \item nella convoluzione, $x_2 \left[ n\right]$ viene shiftata di $-m$, appunto $x_2 \left[ n - m\right]$ 
    \item nella correlazione, $x_2 \left[ n\right]$ viene shiftata di $+m$, appunto $x_2 \left[ n + m\right]$
\end{itemize}

Quindi, proprio per questo, considereremo convoluzione e correlazioni uguali a meno di un segno negativo. \newline 

Come scritto a \ref{eq: DFT tra due sequenze di lunghezza diversa} \nameref{eq: DFT tra due sequenze di lunghezza diversa} a pag \pageref{eq: DFT tra due sequenze di lunghezza diversa}, 
date due sequenze aperiodiche di lunghezza diversa, ad esempio $x_1 \left[ n\right]$ lunga L e $x_2 \left[ n\right]$ lunga P, 
è possibile calcolare la convoluzione ciclica lunga $L+P-1$ perchè la convoluzione ciclica risulta uguale alla convoluzione lineare se calcolata per $L+P-1$ punti. \newline 

Facendo zero-padding alle due sequenze $x_1 \left[ n\right]$ e $x_2 \left[ n\right]$ in modo che siano lunghe $L+P-1$, 
allora si può calcolare le rispettive DFT, moltiplicarle nel dominio DFT, fare l'anti-DFT per poi calcolare la convoluzione lineare. \newline 

Per calcolare le rispettive DFT, si può utilizzare l'algoritmo FFT. \newline 

\newpage

\section{Block convolution}

\footnote{ 
    Slide del prof | 8 - FFT 2 | pag 22\\
    Appunti| 2025-11-05 | pag 2
}

Date due due sequenze, ad esempio:

\begin{itemize}
    \item $x \left[ n\right]$ una sequenza di ingresso ad un filtro  
    \item $h \left[ n\right]$ la risposta di un filtro 
\end{itemize}

e si voglia svolgere una convoluzione lineare tra le due sequenze: 

{
    \Large 
    \begin{equation}
        y \left[ n\right] 
        = 
        x \left[ n\right]
        * 
        h \left[ n\right]
    \end{equation}
}

dove: 

\begin{itemize}
    \item $x \left[ n\right]$ ha lunghezza molto lunga (che tende ad infinito)
    \item $h \left[ n\right]$ ha lunghezza finita (molto più piccola a $x \left[ n\right]$)
\end{itemize}

allora, è necessario applicare una convoluzione a blocchi di durata finita estratti dalla sequenza più lunga. \newline 

Questo tipo di convoluzione prende il nome di block convolution, cioè convoluzione a blocchi in italiano. \newline 

I due metodi più importanti per calcolare una block convolution sono i seguenti: 

\begin{itemize}
    \item overlap and add 
    \item overlap and save 
\end{itemize}

\newpage 

\subsection{Overlap and add}

\footnote{ 
    Slide del prof | 8 - FFT 2 | pag 22 - 23\\
    Appunti| 2025-11-05 | pag 2 \\ 
    Latex | 2025-11-10 Mattino | pag 218 
} 

Data una sequenza $x \left[ n\right]$ molto lunga, si decompone $x \left[ n\right]$ senza sovrapposizione. \newline 

Cioè, dal punto di vista matematico, siccome vorremmo fare: 

{
    \Large 
    \begin{equation}
        y \left[ n\right] 
        = 
        x \left[ n\right]
        *
        h \left[ n\right]
    \end{equation}
}

allora scriviamo $x \left[ n\right]$ come: 

{
    \Large 
    \begin{equation}
        x \left[ n\right]
        = 
        \sum_{r = 0}^{+ \infty}
        x_r \left[ n - rL\right]
    \end{equation}
}

dove: 

\begin{itemize}
    \item $x_r \left[ n \right]$ è la sequenza $x \left[ n\right]$ divisia in L 
    \item L è la lunghezza per cui è stata divisa $x \left[ n\right]$
    \item per $x_r \left[ n - rL\right]$ si intende che la sequenza $x \left[ n\right]$ si divide per ogni multiplo di L
\end{itemize}

L'uscita dal filtro $y \left[ n\right]$, per lo stesso principio di suddivisione, sarà uguale a: 

{
    \Large 
    \begin{equation}
        \begin{split}
        y \left[ n\right]
        &= 
        \sum_{r = 0}^{+ \infty}
        y_r \left[ n - rL\right]
        \\
        &\downarrow
        \\
        y \left[ n\right]
        &= 
        \sum_{r = 0}^{+ \infty}
        x_r \left[ n - rL\right]
        * 
        h \left[ n\right]    
        \end{split}
    \end{equation}
} 

dove: 

\begin{itemize}
    \item $x_r \left[ n - rL\right]$ è lunga di L 
    \item $h \left[ n\right]$ è lunga P 
    \item $y_r \left[ n - rL\right]$ è lunga $P+L-1$, che è proprio la lunghezza della convoluzione lineare tra $x_r \left[ n - rL\right]$ e $h \left[ n\right]$ 
\end{itemize}
