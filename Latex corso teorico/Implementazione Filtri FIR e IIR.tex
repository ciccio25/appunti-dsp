\section{Signal Flow Graph}

\footnote{

} 

Come studiato ad Elettrotecnica, useremo i grafi per rappresentare le equazioni, quindi i circuiti. \newline 

Ma, a differenza di Elettrotecnica dove i grafi sono bidirezionali, cioè lo stesso ramo può essere percorso in tutti i due i versi, 
negli SFG (Signal Flow Graph) i grafi sono unidirezionali. \newline 

Non solo: ad Elettrotecnica ogni ramo conteneva due informazioni, cioè tensione e corrente, invece qui ogni ramo contiene solo un'informazione. \newline 

Gli SFG, generalmente, sono rappresentati nel dominio Z e le operazioni sono implicitamente istantanee. \newline 


Ricordando da \ref{sec: Proprietà della trasformata-z} \nameref{sec: Proprietà della trasformata-z} a pag \pageref{sec: Proprietà della trasformata-z}, 
che, data una sequenza $x \left[ n\right]$ ed un ritardo $n_0$, la sua rispettiva trasformata z vale: 

{
    \Large 
    \begin{equation}
        x \left[ n - n_0\right] 
        \leftrightarrow
        z^{- n_0} X \left( z\right)
    \end{equation}
}

Se il ritardo $n_0$ vale: 

{
    \Large 
    \begin{equation}
        n_0 = 1
    \end{equation}
}

allora avremo che, l'elemento di ritardo nel dominio n è equivalente al prodtto per $z^{-1}$. \newline 

Infatti: 

{
    \Large 
    \begin{equation}
        y \left[ n\right] 
        = 
        x \left[ n - 1\right] 
        \leftrightarrow
        Y \left(z\right)
        =
        z^{- 1} X \left( z\right)
    \end{equation}
}

Quindi, gli elementi che comporranno negli SFG saranno: 

\begin{itemize}
    \item somma di variabili 
    \item prodotto per costante (l'arco sarà un ramo pesato), eventualmente per un ritardo $z^{-n_0}$
\end{itemize}

\section{Strutture per filtri IIR} \label{sec: Strutture per filtri IIR}

\footnote{ 
    Slide del prof | 11 - Implementazione Filtri FIR e IIR 1 | pag 11 - 16, pag 8\\
    Appunti | 2025-11-25 | pag 2 - 3  
} 

Come scritto precedentemente, siccome si considerano gli SFG (Signal Flow Graph) come strutture instantaneee, 
allora, la condizione necessaria per gli SFG è che ci sia almeno un ciclo di ritardo. \newline 

Usualmente, per l'uscita di un filtro $y \left[ n\right]$ si usa la seguente convenzione nel dominio n: 

{
    \Large 
    \begin{equation}
            y \left[ n\right] 
            = 
            \sum_{k = 1}^{N} 
            a_k \cdot y \left[ n - k\right]
            +
            \sum_{k = 0}^{N} 
            b_k \cdot x \left[ n - k\right]
    \end{equation}
}

in cui, nel dominio z, l'SFG, quindi il filtro $H \left(z\right)$, sarà rappresentato nella seguente maniera: 

{
    \Large 
    \begin{equation}
        H \left[z\right]
        = 
        \dfrac{
            \sum_{k = 0}^{N} 
            b_k \cdot z^{-k}
        }{
            1 
            -
            \sum_{k = 1}^{N} 
            a_k \cdot z^{-k}
        } 
        \text{ con } 
        a_0 = 1
    \end{equation}
}

Esistono diverse forme di struttura SFG per rappresentare dei filtri IIR: di seguito un elenco di quelle principali. \newline 

\textbf{Forma diretta I} \newline 

\begin{figure}[h]
    \centering
    \includegraphics[scale = 0.45]{Forma Diretta I filtri IIR.png}
\end{figure}

Possiamo svolgere due osservazioni in merito alla seguente struttura: 

\begin{itemize}
    \item tutte le maglie hanno almeno un elemento di ritardo (cioè $z^{-1}$) 
    \item la forma diretta realizza prima gli zeri e poi i poli 
    \item l'SFG è uni-direzionale, parte da sinistra con la sequenza di ingrezzo $x \left[ n\right]$ ed ha in uscita, a destra dell'SFG, la sequenza $y \left[ n\right]$
\end{itemize} 

Grazie alla proprietà di trasposizione degli SFG (cioè che si può scambiare l'ingresso con l'uscita e viceversa e rovesciare tutte le frecce degli archi senza alterare la $H \left(z\right)$),

\textbf{Forma in cascata} \newline 

La funzione di rete può essere fattorizzata e riscritta come segue: 

{
    \Large 
    \begin{equation}
        H \left(z\right) 
        = 
        A \cdot 
        \dfrac{
            \prod_{k = 1}^{M_1} 
            \left(1 - c_{k}^{'} z^{-1}\right)
            \prod_{k = 1}^{M_2} 
            \left(1 - c_{k}^{''} z^{-1}\right)
            \left(1 - c_{k}^{''*} z^{-1}\right)
        }{
            \prod_{k = 1}^{N_1} 
            \left(1 - d_{k}^{'} z^{-1}\right)
            \prod_{k = 1}^{N_2} 
            \left(1 - d_{k}^{''} z^{-1}\right)
            \left(1 - d_{k}^{''*} z^{-1}\right)
        }
    \end{equation}
}

dove: 

\begin{itemize}
    \item $ \left(1 - d_{k}^{''*} z^{-1}\right)$ è il complesso coniugato di $\left(1 - d_{k}^{''} z^{-1}\right)$ 
    \item $\left(1 - c_{k}^{''*} z^{-1}\right)$ è il complesso coniugato di $\left(1 - c_{k}^{''} z^{-1}\right)$
\end{itemize}

Scrivere in questa forma $H \left(z\right)$ permette di avere i seguenti vantaggi: 

\begin{itemize}
    \item libertà realizzativa, in particolare sulla scelta delle sezioni e dell'ordine 
    \item robustezza numerica 
\end{itemize}

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 2}
Robustezza numerica perchè, essendoci tanti termini in $H \left(z\right)$, il singolo elemento che compone la funzione di rete influisce poco su tutta la funziona di rete? \newline 

Quindi, di conseguenza, avremo una robustezza numerica? 
\end{tcolorbox} 

Ma, in generale, se si realizza una forma in cascata, si cerca di utilizzerare non termini del primo ordine, cioè $z^{-1}$, 
bensì $z^{-2}$, come si nota nella seguente formulazione di $H \left(z\right)$: 

{
    \Large 
    \begin{equation}
        H \left(z\right) 
        = 
        \prod_{k = 1}^{N_s} 
        \dfrac{
            b_{0k} + b_{1k} z^{-1} + b_{2k} z^{-1}
        }{
            1 - a_{1k} z^{-1} + a_{2k} z^{-1}
        }
    \end{equation}
}

in cui: 

{
    \Large 
    \begin{equation}
        N_s = \text{trunc} \left(\dfrac{N + 1}{2}\right)
    \end{equation}
} 

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 3}
"trunc" perchè si prende la parte intera della frazione? \newline 

Inoltre, si prende la metà grazie alle proprietà di simmetria di $H \left(z\right)$? 
\end{tcolorbox}

\textbf{Forma parallela} \newline 

La funzione di rete può essere sviluppata in frazioni parziali: 

{
    \Large 
    \begin{equation}
        H \left(z\right) 
        = 
        \sum_{k = 0}^{M - N}
        C_k z^{-k} 
        + 
        \sum_{k = 1}^{N_1} 
        \dfrac{A_k}{1 - d_{k}^{'} z^{-1}}
        +
        \sum_{k = 1}^{N_1} 
        \dfrac{B_k \left(1 - e_k z^{-1}\right)}{\left( 1 - d_{k}^{''} z^{-1}\right)\left( 1 - d_{k}^{''*} z^{-1}\right)} 
    \end{equation}
}

dove: 

\begin{itemize}
    \item $M \geq N$ 
    \item $h \left[ n\right]$ è reale, ha poli semplici e complessi coniugati
\end{itemize}

Usualmente, nella forma parallela, si utilizza termini del secondo ordine. \newline 

Questo tipo di forma, permette di avere i seguenti vantaggi: 

\begin{itemize}
    \item libertà realizzativa, in particolare la scelta delle sezioni 
    \item è parallelizzabile, quindi molto utili, ad esempio, nelle schede grafiche 
\end{itemize}

\footnote{

} 

I filtri FIR, cioè i filtri a risposta impulsiva finita, sono casi particolari delle strutture IIR, 
ma più importanti nella pratica. \newline 

Quindi passeremo da: 

{
    \Large 
    \begin{equation}
        H \left[z\right]
        = 
        \dfrac{
            \sum_{k = 0}^{N} 
            b_k \cdot z^{-k}
        }{
            1 
            -
            \sum_{k = 1}^{N} 
            a_k \cdot z^{-k}
        } 
        \text{ con } 
        a_0 = 1
    \end{equation}
} 

al caso di $H \left(z\right)$ per i FIR che vale: 

{
    \Large 
    \begin{equation}
        H \left(z\right) 
        = 
        \sum_{k = 0}^{M} 
        b_k \cdot z^{-k}
    \end{equation}
}

I FIR, rispetto agli IIR, sono dei filtri in cui la media è pesata (cioè in inglese moving average). \newline

Osservando le seguenti strutture dei filtri FIR, possiamo svolgere le seguenti considerazioni: 

\begin{itemize}
    \item rispetto ai filtri IIR, la forma parallela non esiste per i filtri FIR perchè, non essendoci i poli, non è possibile applicare il teorema dei residui 
    \item gli SFG dei filtri FIR non cotengono cicli o frecce in direzione opposte, e questa è la condizione sufficiente per un SFG di un FIR
\end{itemize} 

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 4} 
    "gli SFG dei filtri FIR non cotengono cicli o frecce in direzione opposte" cioè nei FIR non ci sono le retroazioni, giusto? 
\end{tcolorbox} 

Non ci occupiamo quasi mai della fase, ma la condizione di simmetria o anti-simmetria dei coefficienti è sufficiente per avere fase lineare generalizzare, cioè ritardo di gruppo costante. \newline 

Questa simmetria può essere sfruttata per semplificare la struttura dei filtri con un conseguente risparmio di circa la metà dei moltiplicatori, cioè dei k-esimi coefficienti $b_k$. \newline 


Ad esempio, per un fitro FIR I, che avrà questa formulazione matematica: 

{
    \Large 
    \begin{equation}
        y \left[ n\right] 
        = 
        \sum_{k = 0}^{\frac{M}{2} - 1} 
        b_k \left[ x  \left[ n-k\right] + x \left[n - M + k \right]\right]
        + 
        b_{\frac{M}{2}} x  \left[ n- \dfrac{M}{2}\right] 
    \end{equation}
}

{

\section{Implementazioni digitali}

\footnote{ 
    Slide del prof | 11 - Implementazione Filtri FIR e IIR 2 | pag 2 - 6\\

} 

Le strutture viste in precedenza vengono usualmente implementate mediante circuiti digitali, 
progettati appositamente o programmabili, e presentano, pertanto, problemi di quantizzazione, che comporta ad una precisione finita. \newline 

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 5} 
"circuiti digitali programmabili" ad esempio attraverso gli FPGA? 
\end{tcolorbox} 

Avevamo già affrontato i problemi di quantizzazione precedentemente in \ref{subsec: Conversione D/A reale} \nameref{subsec: Conversione D/A reale} a pag \pageref{subsec: Conversione D/A reale}. \newline 

In parole povere (e senza ritornare indietro nelle pagine precedenti), i problemi di quantizzazione esistono perchè nel mondo reale la grandezza fisica, come ad esempio una tensione, ha infiniti valori, 
invece un circuito digitale, grazie alla quantizzazione, può interpretare le grandezze fisiche reali attraverso una sua rappresentazione finita, e quindi con finiti valori: da ciò nasce un errore di quantizzazione, che è la differenza tra il valore reale e il valore quantizzato. \newline 

\begin{tcolorbox}
    Prima di approfondire il tema della rappresentazioni dei numeri nei circuiti digitali, ti consiglio di guardare i seguenti video: 

    \begin{itemize}
        \item computers suck at division (a painful discovery) by Low Level \\\url{https://youtu.be/ssDBqQ5f5_0?si=NfVkwva6Uww6qjaz} 
        \item How Computers perform Division by Serious Electrical Engineering \\\url{https://youtu.be/WyBsPfhlC2M?si=tTkUh6_f_3fzq_31}
    \end{itemize}

    Perchè vedere dei video sulle divisioni? Perchè è il caso più complicato che un calcolatore possa compiere (e non è una cosa scontata come pensi). 
\end{tcolorbox} 

Il circuito digitale può rappresentare il valore quatizzato con due tipi di rappresentazione: 

\begin{itemize}
    \item virgola mobile (in inglese floating point) 
    \item virgola fissa (in inglese fixed point)
\end{itemize} 

Il caso più complicato è quello della rappresentazione in virgola movile perchè vengono utilizzati tre elementi per rappresentare un numero: 

\begin{itemize}
    \item segno 
    \item mantissa 
    \item esponente 
\end{itemize}

Per quanto riguarda la quantizzazione, mantiene una grande dinamica ed un piccolo rumore di quantizzazione. \newline 

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 6} 
Cosa si intende per "grande dinamica"? Cioè con pochi elementi (cioè segno, mantissa ed esponente) si possono rappresentare tanti valori? 
\end{tcolorbox}  

L'IEEE (l'ente standardizzatore di riferimento), grazie allo standard IEEE 754, permette, ad esempio, i seguenti formati in virgola mobile: 

\begin{itemize}
    \item 32 bit, o anche chiamata come singola precisione 
    \item 64 bit, o anche chiamata come dobbia precisione 
\end{itemize} 

\newpage 

Come si visualizza dalla seguente schermata: 

\begin{figure}[h]
    \centering
    \includegraphics[scale = 1.5]{IEEE754.png}
\end{figure} 

il formato a 64 bit, cioè quello a doppia precisione, ha un errore minore rispetto al formato a 32 bit, cioè quello a singola precisione, 
al costo di gestire più bit. \newline 

Invece, al posto di utilizzare una rappresentazione a virgola mobile, si può utilizzare un formato a virgola fissa (in inglese fixed point). \newline 

Il formato a virgola fissa è meno costoso e più veloce rispetto al formato a virgola mobile ed è formato dai seguenti elementi: 

\begin{itemize}
    \item segno 
    \item grandezza 
    \item complemento 
\end{itemize}

A sua volta, il complemento può essere a: 

\begin{itemize}
    \item complemento a 1 (in inglese 1's complement) 
    \item complemento a 2 (in inglese 2's complement), che è il formato più utilizzato 
\end{itemize} 

Per un numero rappresentato con B+1 bit, la rappresentazione complemento a 2 risulta: 

{
    \Large 
    \begin{equation}
        \setlength{\jot}{10pt} % affecting the line spacing in the environment
        \begin{split}
            \hat{x} \left[ n\right] 
            &= 
            Q_{B} \left[ x\right] 
            \\
            &= 
            X_{m} 
            \cdot 
            \left(
                -b_{0} 
                + 
                \sum_{i = 1}^{B} 
                b_{i} 
                \cdot 
                2^{-i}
            \right) 
            \\
            &= 
            X_{m} \cdot \hat{x}_{B}
        \end{split}
    \end{equation}
}

dove: 

\begin{itemize}
    \item $\hat{x} \left[ n\right]$ è la sequenza contenente i valori quantizzati 
    \item $Q_{B} \left[ x\right]$ è l'operatore di quantizzazione 
    \item $X_{m}$ è il valore massimo della sequenza quantizzata $x \left[ n\right]$ 
    \item $-b_{0}$ è il segno della sequenza, quindi se positiva o negativa 
    \item $\hat{x}_{B}$ è il b-esimo bit della sequenza $\hat{x} \left[ n\right] $ 
\end{itemize}

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 7} 
"$X_{m}$ è il valore massimo della sequenza quantizzata $x \left[ n\right]$", giusto? \newline 

$x \left[ n\right]$ possiamo considerarla la sequenza campionata? \newline 

"$\hat{x}_{B}$ è il b-esimo bit della sequenza $\hat{x} \left[ n\right] $", giusto? 
\end{tcolorbox}  

Se consideriamo la sequenza $\hat{x} \left[ n\right]$ compresa tra due massimi, il cui valore assoluto $\abs{X_{M}}$ è uguale: 

{
    \Large 
    \begin{equation}
        - X_{M} \leq \hat{x} \left[ n\right] \leq + X_{M}
    \end{equation}
}

in cui il passo $\Delta$, cioè la distanza tra un valore compionato e l'altro, la consideriamo come: 

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 8} 
"il passo $\Delta$, cioè la distanza tra un valore compionato e l'altro" è giusto? 
\end{tcolorbox}  

{
    \Large 
    \begin{equation}
        \Delta = X_{m} \cdot 2^{-B}
    \end{equation}
}

allora avremo che l'errore è uguale a: 

{
    \Large 
    \begin{equation}
        e = Q_{b} \left[ x\right] - x \left[ n\right]
    \end{equation}
} 

in cui l'arrotondamento è uguale a: 

{
    \Large 
    \begin{equation}
        - \dfrac{\Delta}{2} 
        < 
        e
        <
        + \dfrac{\Delta}{2} 
    \end{equation}
}

invece il troncamento vale: 

{
    \Large 
    \begin{equation}
        -\Delta 
        < 
        e
        <
        0 
    \end{equation}
}

Dal punto di vista grafico, la funzione arrotondamento avrà il seguente andamento:

\subsection{Quantizzazione dei coefficienti}

\footnote{ 
    Slide del prof | 11 - Implementazione Filtri FIR e IIR 2 | pag 7 - 8\\
    Appunti mancanti   
} 

Considerando un circuito IIR generico, avremo che la funzione di rete $H \left(z\right)$ è uguale a: 

{
    \Large 
    \begin{equation}
        H \left(z\right) 
        = 
        \dfrac{
            \sum_{k = 0}^{M}
            b_k \cdot z^{-k}
        }{
            1 - \sum_{k = 1}^{N} a_k \cdot z^{-1} 
        }
    \end{equation}
}

in cui la precisione del filtro è infinita. \newline 

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 18} 
"Precisione infinita" perchè il filtro si estende nel tempo fino ad infinito? $+ \infty$
\end{tcolorbox} 

Invece, considerando i coefficienti quantizzati, avremo che la nuova funzione di rete non sarà $H \left(z\right)$, bensì: 

{
    \Large 
    \begin{equation}
        \hat{H} \left(z\right) 
        = 
        \dfrac{
            \sum_{k = 0}^{M}
            \hat{b_k} \cdot z^{-k}
        }{
            1 - \sum_{k = 1}^{N} \hat{a_k} \cdot z^{-1} 
        }
    \end{equation}
}

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black] \label{q: 19} 
Perchè abbiamo dei coefficienti quantizzati in un filtro? 
\end{tcolorbox} 

Kaiser nel 1966 ha dimostrato che, a causa della quantizzazione, i poli e gli zeri del sistema $H \left(z\right)$ si spostano. \newline 

Proprio perchè i poli e gli zeri di $\hat{H} \left(z\right)$ sono quelli di $H \left(z\right) $ ma spostati, allora la risposta del circuito totale cambia. \newline
